# FileShare文件共享系统

核心需求：开发适配10-50人小团队的文件共享系统，需支持私有服务器部署，重点实现文件集中管理、精准权限控制、多端访问，兼顾安全与易用性，参考Zoho WorkDrive、Nextcloud核心优势并简化冗余功能。
具体开发指令如下：

## 1. 基础部署与访问

1. 支持部署在Linux（CentOS、Ubuntu）及Windows Server私有服务器，无公有云依赖；
2. 适配多端访问：电脑端（Windows/Mac桌面客户端+网页端）、手机端（iOS/Android APP），支持断网编辑后联网自动同步；
3. 基础支持账号密码登录，预留企业微信/钉钉第三方登录接口。

## 2. 文件存储与管理

1. 实现分层空间管理：团队公共空间、部门空间、个人空间、项目专属空间，支持多级文件夹创建；
2. 支持单个/批量文件上传（单文件最大≥10GB，支持断点续传）、下载、复制、移动、重命名、删除；
3. 自动保存文件历史版本（标注修改人/时间，默认保留≥30天，支持一键恢复）；
4. 支持管理员配置文件命名规则、回收站保留周期，防止误删与命名混乱。

## 3. 核心权限控制

1. 预设四级角色权限：管理员（全权限，含用户/权限/系统配置/日志审计）、编辑者（上传/修改/删除有权限文件）、查看者（仅查看/下载）、仅评论者（仅查看/评论）；
2. 支持按文件夹维度对单个用户/用户组精准授权，敏感文件夹可单独配置权限；
3. 外部分享默认关闭，开启后可设置指定邮箱访问、密码、有效期（7天/30天/自定义），支持查看访问记录与一键撤销；
4. 支持管理员动态调整权限、离职成员权限快速撤销。

## 4. 协作与检索

1. 支持文档、表格、PPT、图片、PDF等常见格式文件在线预览；
2. 支持文件添加评论、@成员提醒；
3. 实现多条件组合检索（文件名/类型/修改时间/上传人）+关键词模糊搜索，结果按相关性排序。

## 5. 性能与安全要求

1. 性能：支持≥30人同时在线，单文件夹1000+文件加载≤3秒，服务器内存占用≤2GB、CPU占用≤30%，低配服务器稳定运行；
2. 安全：文件传输HTTPS加密、存储AES-256加密，详细记录操作日志（含操作人/时间/内容/IP，保留≥90天可导出），支持“禁止下载”权限配置防止泄露。

## 6. 易用性与运维

1. 界面简洁，核心功能（上传/共享/权限设置）入口明显，新手10分钟内掌握基础操作，操作失败给出明确中文提示；
2. 提供可视化管理员后台：支持用户管理、权限批量配置、存储空间监控、日志查看；
3. 支持手动/自动备份（可配置周期），故障时快速恢复数据；
4. 预留存储空间扩容、第三方工具（项目管理/OA）集成接口。

## 7. 额外要求

1. 简化大型企业冗余功能（如复杂审批、高级报表），聚焦小团队轻量化需求；
2. 配套简易操作手册（图文教程），覆盖核心操作，支持基础部署指导与故障排查。
